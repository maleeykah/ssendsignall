# Forms
This defines how forms behave across the app.

## Example scope:

- form layout
- label placement
- validation messaging
- required fields
- error display
- submit states

## Form rules and patterns
- All forms must include labels.
- Required fields must be marked clearly.
- Validation must occur server-side before persistence.
- Error messages must appear below the field.
- Submit buttons must show loading states during submission.
